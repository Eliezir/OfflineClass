Apresenta.AI

App desktop que transforma conteúdo livre em apresentações HTML interativas geradas por IA. Você escreve, a IA estrutura em Markdown, você refina no chat, escolhe um modelo visual — e a IA gera a apresentação final em HTML real, com animações e transições. Este guia apresenta o produto, a arquitetura e como contribuir.

Electron + React TypeScript ponta a ponta Hono + Prisma Anthropic Claude Monorepo pnpm Projeto POO · IFAL

O produto

Como funciona

O fluxo central, do texto livre à apresentação pronta. A IA aparece em duas etapas: estruturar o conteúdo e gerar o HTML final.

Decisões fechadas: app local de usuário único (sem login); o onboarding é opcional e começa no provedor Mock; o editor de conteúdo e de roteiro é o mesmo Markdown (Monaco); o apresentador roda em janela Electron dedicada, sem chrome. O BCrypt existe só para proteger a API key dos provedores de IA salva no SQLite.

Backend

API em camadas (MVC)

Migrado de Java/Spring Boot para TypeScript mantendo a mesma arquitetura: uma requisição desce por Controller → BO → DAO → Models e nunca pula etapas. A composição (DI manual) vive em src/app.ts.

Controllersrc/controllers/

Camada HTTP (routers Hono), montada em /api/*. Valida entrada e delega ao BO.

BOsrc/bo/

Regra de negócio. Depende só do DAO; mapeia entidades para VOs.

DAOsrc/dao/

Acesso a dados; envolve o PrismaClient.

Models / DBprisma/schema.prisma

Entidades + SQLite. DTOs imutáveis como schemas Zod em src/models/vo/.

🧱

Camadas garantidas

dependency-cruiser reprova no CI atalhos entre camadas (substitui o ArchUnit).

🚨

Contrato de erro

Tudo vira { timestamp, status, erro, mensagens[] } via app.onError.

🗄️

Migrations

Tutorial · backend

Criando um endpoint do zero

Vamos adicionar um recurso completo — Tarefa — atravessando todas as camadas, com teste primeiro. Mesmo sem conhecer as ferramentas, siga os passos: cada um linka a documentação oficial. A fatia exemplo (em backend/src/**/exemplo.*) é a referência viva desse padrão.

💡 Ordem TDD: para cada camada, escreva o teste que falha, deixe o Vitest rodando com pnpm test:watch e só então implemente até ficar verde. O que é TDD?

Modele a entidade no schema do Prisma

As entidades ficam em prisma/schema.prisma — Prisma Schema.

model Tarefa {
  id        Int     @id @default(autoincrement())
  titulo    String
  concluida Boolean @default(false)
}

Gere e aplique a migration (Prisma Migrate):

pnpm migrate:dev --name cria_tarefa

Isso cria prisma/migrations/<timestamp>_cria_tarefa/ e regenera o client tipado.

Defina os DTOs com Zod

Em src/models/vo/tarefa.vo.ts — o schema Zod é a validação e o tipo.

// src/models/vo/tarefa.vo.ts
import { z } from 'zod'

// entrada do POST (validação + tipo inferido)
export const criaTarefaSchema = z.object({
  titulo: z.string().min(1, 'O título é obrigatório')
})
export type CriaTarefa = z.infer<typeof criaTarefaSchema>

// formato de resposta da API
export type TarefaVO = { id: number; titulo: string; concluida: boolean }

DAO — acesso a dados (teste → código)

Escreva o teste com o banco de teste real, depois implemente o DAO sobre o Prisma Client.

// test/tarefa.dao.test.ts
import { describe, expect, it } from 'vitest'
import { prisma } from '../src/config/prisma'
import { TarefaDao } from '../src/dao/tarefa.dao'

describe('TarefaDao', () => {
  it('cria e lista tarefas', async () => {
    const dao = new TarefaDao(prisma)
    await dao.criar('Estudar Hono')
    const todas = await dao.listar()
    expect(todas).toHaveLength(1)
  })
})
// src/dao/tarefa.dao.ts
import type { PrismaClient, Tarefa } from '@prisma/client'

export class TarefaDao {
  constructor(private readonly prisma: PrismaClient) {}

  listar(): Promise<Tarefa[]> {
    return this.prisma.tarefa.findMany({ orderBy: { id: 'asc' } })
  }

  criar(titulo: string): Promise<Tarefa> {
    return this.prisma.tarefa.create({ data: { titulo } })
  }
}

BO — regra de negócio

O BO depende só do DAO (injetado no construtor) e devolve VOs. No teste, injete um DAO falso.

// test/tarefa.bo.test.ts
import { describe, expect, it } from 'vitest'
import { TarefaBo } from '../src/bo/tarefa.bo'
import type { TarefaDao } from '../src/dao/tarefa.dao'

describe('TarefaBo', () => {
  it('mapeia a entidade para VO', async () => {
    const fakeDao = {
      criar: () => Promise.resolve({ id: 1, titulo: 'X', concluida: false })
    } as unknown as TarefaDao
    expect(await new TarefaBo(fakeDao).criar({ titulo: 'X' }))
      .toEqual({ id: 1, titulo: 'X', concluida: false })
  })
})
// src/bo/tarefa.bo.ts
import type { TarefaDao } from '../dao/tarefa.dao'
import type { CriaTarefa, TarefaVO } from '../models/vo/tarefa.vo'

export class TarefaBo {
  constructor(private readonly dao: TarefaDao) {}

  async listar(): Promise<TarefaVO[]> {
    const tarefas = await this.dao.listar()
    return tarefas.map((t) => ({ id: t.id, titulo: t.titulo, concluida: t.concluida }))
  }

  async criar(input: CriaTarefa): Promise<TarefaVO> {
    const t = await this.dao.criar(input.titulo)
    return { id: t.id, titulo: t.titulo, concluida: t.concluida }
  }
}

Controller — camada HTTP + validação

Um router Hono. O validar (veja Receitas) faz a validação cair no contrato de erro.

// src/controllers/tarefa.controller.ts
import { Hono } from 'hono'
import { validar } from '../config/validar'
import { criaTarefaSchema } from '../models/vo/tarefa.vo'
import type { TarefaBo } from '../bo/tarefa.bo'

export function tarefaController(bo: TarefaBo): Hono {
  const router = new Hono()

  router.get('/', async (c) => c.json(await bo.listar()))

  router.post('/', validar('json', criaTarefaSchema), async (c) => {
    const dados = c.req.valid('json')
    return c.json(await bo.criar(dados), 201)
  })

  return router
}

Ligue tudo no app.ts (injeção de dependência)

No composition root, instancie a cadeia e monte o router em /api/tarefas.

// src/app.ts — dentro de createApp(prisma)
const tarefaDao = new TarefaDao(prisma)
const tarefaBo = new TarefaBo(tarefaDao)
app.route('/api/tarefas', tarefaController(tarefaBo))

Teste o endpoint de ponta a ponta

Sem subir servidor: o Hono testa em processo com app.request().

// test/tarefa.controller.test.ts
import { describe, expect, it } from 'vitest'
import { prisma } from '../src/config/prisma'
import { createApp } from '../src/app'

describe('/api/tarefas', () => {
  it('POST cria (201) e GET lista', async () => {
    const app = createApp(prisma)
    const r = await app.request('/api/tarefas', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ titulo: 'Estudar Prisma' })
    })
    expect(r.status).toBe(201)
    const lista = await app.request('/api/tarefas')
    expect(await lista.json()).toHaveLength(1)
  })

  it('POST sem título → 400 no contrato de erro', async () => {
    const app = createApp(prisma)
    const r = await app.request('/api/tarefas', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({})
    })
    expect(r.status).toBe(400)
  })
})

Receitas práticas

Tarefas comuns do dia a dia, com o código mínimo.

🧾 Validar entrada (Zod)

Crie um helper que joga a falha de validação no error-handler:

// src/config/validar.ts
import { zValidator } from '@hono/zod-validator'
import type { ZodSchema } from 'zod'

type Alvo = 'json' | 'query' | 'param'

export const validar = (alvo: Alvo, schema: ZodSchema) =>
  zValidator(alvo, schema, (r) => {
    // vira ErroResponse 400 automaticamente
    if (!r.success) throw r.error
  })

@hono/zod-validator

🚨 Lançar erros de negócio

Lance exceções; o error-handler (em app.onError) cuida do status e do corpo.

import { ExcecaoNaoEncontrado } from '../exception/excecao-nao-encontrado'

const tarefa = await this.dao.buscar(id)
if (!tarefa) {
  throw new ExcecaoNaoEncontrado('Tarefa ' + id + ' não existe')
}
// → 404 { timestamp, status, erro, mensagens }

ExcecaoNaoEncontrado → 404 · ExcecaoValidacao → 400 · ExcecaoApp → 500.
